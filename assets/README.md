# Assets フォルダ

このフォルダには、Lucid-SRD販売サイトで使用する画像・動画ファイルを配置します。

## フォルダ構造

```
assets/
├── images/           # 画像ファイル
│   ├── favicon.png   # ファビコン（推奨: 32x32 または 64x64）
│   ├── logo.png      # ロゴ画像
│   ├── hero-bg.jpg   # ヒーローセクション背景画像
│   ├── use-case-1.jpg  # ユースケース画像1（顎変形症）
│   ├── use-case-2.jpg  # ユースケース画像2（解剖学教育）
│   ├── use-case-3.jpg  # ユースケース画像3（症例検討）
│   ├── ogp-image.jpg   # OGP用画像（推奨: 1200x630）
│   └── twitter-image.jpg  # Twitter Card用画像（推奨: 1200x600）
│
└── videos/           # 動画ファイル
    └── hero-video.mp4  # ヒーローセクション背景動画（オプション）
```

## 画像の推奨仕様

### ファビコン (favicon.png)
- サイズ: 32x32px または 64x64px
- フォーマット: PNG
- 背景: 透過または単色

### ロゴ (logo.png)
- サイズ: 高さ 50-80px 程度
- フォーマット: PNG（透過推奨）
- カラー: 白色版とカラー版の両方を用意することを推奨

### ヒーロー背景画像 (hero-bg.jpg)
- サイズ: 1920x1080px 以上（高解像度推奨）
- フォーマット: JPG または WebP
- ファイルサイズ: 500KB以下（最適化推奨）
- 内容: Sony Spatial Reality Display、医療3D画像、手術シーン等

### ユースケース画像 (use-case-1.jpg, use-case-2.jpg, use-case-3.jpg)
- サイズ: 640x480px 程度
- フォーマット: JPG または WebP
- ファイルサイズ: 各200KB以下
- 内容:
  1. 顎変形症の手術シミュレーション画像
  2. 解剖学教育での活用画像
  3. 症例検討での活用画像

### OGP画像 (ogp-image.jpg)
- サイズ: 1200x630px（Facebook推奨サイズ）
- フォーマット: JPG
- ファイルサイズ: 300KB以下
- 内容: 製品名、キャッチコピー、製品画像を含むビジュアル

### Twitter Card画像 (twitter-image.jpg)
- サイズ: 1200x600px
- フォーマット: JPG
- ファイルサイズ: 300KB以下
- 内容: OGP画像と同様

## 動画の推奨仕様

### ヒーロー動画 (hero-video.mp4)
- サイズ: 1920x1080px
- フォーマット: MP4 (H.264コーデック)
- ファイルサイズ: 5MB以下推奨
- 長さ: 10-30秒のループ動画
- 音声: なし（ミュート推奨）
- 内容: Sony Spatial Reality Displayでの3D可視化デモ

## 画像最適化の推奨ツール

- **TinyPNG**: https://tinypng.com/ (PNG/JPG圧縮)
- **Squoosh**: https://squoosh.app/ (Google製の画像最適化ツール)
- **ImageOptim**: https://imageoptim.com/ (Mac用の画像最適化アプリ)

## 注意事項

1. **著作権**: すべての画像・動画は使用許諾を得たもの、またはオリジナルのものを使用してください
2. **最適化**: Webサイトのパフォーマンスを保つため、画像は必ず最適化してから配置してください
3. **代替テキスト**: すべての画像に適切な alt 属性を設定してください
4. **レスポンシブ対応**: 複数サイズの画像を用意することで、モバイルでも高速に表示できます
